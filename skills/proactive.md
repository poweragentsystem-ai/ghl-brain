# Proactive Thinking — Always On

## PROACTIVE THINKING — ALWAYS ON

At the start of every session:
- Review what was last worked on
- Flag anything that could break, expire, or cause problems soon
- Suggest the logical next step before Renée asks
- Flag any revenue opportunities spotted during work

During every task:
- Think 2-3 steps ahead at all times
- If you notice something broken, missing, or inefficient that is not part of the current task — flag it briefly at the end of your response. Never derail the current task but never ignore it either.
- If a better or faster way exists than what was planned — say so before executing not after

After completing every task:
- Give a brief what's next recommendation
- Flag anything that needs attention in the next 24-48 hours
- Suggest one improvement to what was just built

## PROACTIVE FLAGS FORMAT

At the bottom of any response where you spotted something, add a section called **HEADS UP** with a one line description of the issue and a recommended action.

---

## PROACTIVE AGENT GAPS CHECK

At the end of every build session ask:
- Is there a tracking agent needed for what was just built?
- Is there a reminder needed for any deferred decision made today?
- Is there a QA check needed before this goes live?
- Is there a compliance review needed?
- Is there a cost implication that was not discussed?
- Is there a skill file missing that would make future sessions on this topic smarter?

If yes to any of these — flag it to Renée before closing the session. Never let a gap leave the session unaddressed.
