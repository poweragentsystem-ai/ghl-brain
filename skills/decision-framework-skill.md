# DECISION FRAMEWORK SKILL
### For: Brian (Strategy), Jordan (COO) — any agent helping Renée choose between options

---

## WHAT THIS SKILL IS

When Renée is weighing two or more options — tools, strategies, offers, platforms, hires, pivots — this skill runs a structured decision analysis and delivers a clear recommendation. Not just a list. A verdict with reasoning.

The difference between a useful decision framework and a useless one: a useless one says "it depends." A useful one picks a side, explains why, and tells Renée what she'd be giving up by choosing the other path.

---

## WHEN TO ACTIVATE THIS SKILL

Trigger this skill any time Renée is:
- Choosing between two platforms, tools, or approaches
- Deciding whether to build vs. buy vs. skip
- Evaluating a business pivot or new revenue stream
- Weighing a hire, contractor, or partnership
- Stuck between two directions and needs a push
- Asking "should I...?" or "which is better...?" or "what do you think about X vs Y?"

---

## THE DECISION FRAMEWORK — 5 LAYERS

Run all 5 layers for every decision. Never skip a layer. Present them in order.

---

### LAYER 1 — CLARIFY THE REAL QUESTION

Before analyzing anything, restate the decision in one sentence. Often what someone asks is not the real question.

Example:
- Asked: "Should I use GHL or n8n for this?"
- Real question: "Which tool gets this live fastest with the least ongoing maintenance?"

State the real question out loud before proceeding. If the real question is different from what was asked, flag it.

---

### LAYER 2 — THE SCORECARD

Score each option across 6 dimensions. Score 1–5 (5 = best).

| Dimension | What it measures |
|---|---|
| **Speed to result** | How fast does this get Renée to the outcome? |
| **Cost** | Total cost — money, time, tokens, complexity |
| **Reversibility** | How easy to undo if it's wrong? |
| **Revenue impact** | Does this make money, save money, or enable revenue? |
| **Maintenance burden** | How much ongoing work does this create? |
| **Fit with current stack** | Does this work with what already exists? |

Total the scores. Higher = better fit.

Present as a simple table. No padding. No softening.

---

### LAYER 3 — PROS AND CONS (REAL ONES ONLY)

For each option, list only the pros and cons that actually matter for THIS decision and THIS business. No generic lists. No filler.

**Format:**
- Pro: [specific, concrete benefit]
- Con: [specific, concrete risk or cost]

Maximum 4 pros and 4 cons per option. If you can't fill 4, don't pad it.

Flag any con that is a dealbreaker — a reason to eliminate the option entirely regardless of its pros.

---

### LAYER 4 — RISK ASSESSMENT

For each option, answer three questions:
1. **What's the worst realistic outcome if this goes wrong?**
2. **How likely is that outcome? (Low / Medium / High)**
3. **Can Renée recover from it? (Yes / With difficulty / No)**

If the worst outcome is unrecoverable — flag it clearly. That changes the recommendation.

---

### LAYER 5 — THE VERDICT

Pick one. Not "it depends." Not "both have merit." One.

Structure:
- **Recommended:** [Option name]
- **Why:** [1-3 sentences — the core reason]
- **What you're giving up:** [The real cost of not choosing the other option — be honest]
- **Condition that would change this:** [The one thing that, if true, flips the recommendation]

If Renée has already decided and is just looking for validation — say so. "You've already decided. Here's why it's the right call / here's what to watch out for."

---

## HOW BRIAN USES THIS SKILL

When Renée brings a decision:
1. Read the question
2. Run Layer 1 — clarify the real question, state it out loud
3. Run the Scorecard (Layer 2)
4. Run Pros/Cons (Layer 3) — real ones only, no padding
5. Run Risk Assessment (Layer 4)
6. Deliver the Verdict (Layer 5) — one option, clear reasoning

Total output should be readable in under 2 minutes. No essays. No hedging.

---

## TONE AND RULES

- Pick a side. Always. Fence-sitting is not analysis — it's abdication.
- Be honest about tradeoffs. Don't make the recommendation look better than it is.
- If both options are bad — say so and suggest a third path.
- If Renée is about to make a mistake — say so directly. "This is the wrong call because X." Then explain.
- Never say "it depends" without immediately saying what it depends on and resolving it.
- Renée's time is the scarcest resource. The answer should be clear enough that she can decide in 60 seconds.

---

## QUICK FORMAT (for simple decisions)

For fast, low-stakes decisions — skip the full framework and use this:

```
REAL QUESTION: [one sentence]

[Option A] — Score: X/30
Best reason to pick it: [one sentence]
Biggest risk: [one sentence]

[Option B] — Score: X/30
Best reason to pick it: [one sentence]
Biggest risk: [one sentence]

VERDICT: [Option] — [one sentence why]
Condition that flips it: [one sentence]
```
